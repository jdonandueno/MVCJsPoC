MVC Comparison

Angular Ember React Riot
Pros Developed & maintained by Google, extense community, two-way data binding Server-side rendering, app out-of-the-box development, industry leading router Easy to prototype, ease of debugging, interactable with other JS plugins Small, lightweight library easy to learn and maintain
Cons Highly complex runtime, scope debugging is difficult and very steep learning curve Small community, difficult to develop outside of its comfort zone Verbose and not a full framework (no router, or model management) Performance issues are often overcome by the size of the framework, but has room for improvement

Other alternatives

Aurelia Backbone Knockout Polymer Vue
Concept MV* Conventions M/V data binding MVVM Web Components made easy Modular Framework
Pros ES6 native compatibility, modular and standalone, compatible with Flux Fast, lightweight framework, for building small solutions, lots of integrated functionality out of the box Highly compatible with 3rd party JS libraries, easy to use, and data binding is lighter thanks to its core structure Relies on native browser compatibility, rather than external JS libraries, maintains a separated view design from functionality Virutal DOM rendering, with higher performance on data bindings, lightweight,
Cons Performance leaves some room to improvement, bundling is non-standard and often mishandled Debugging is extensive and complicated, longer development time and understanding of core library Difficult to organize in larger aplications, dependency errors are prone to happen, lack of server side rendering, and download time of polyfills Data binding is made possible by functiones instead of simple objects, HTML views can get messy on increasing complexity, does not provide routing Slightly new framework, with a lot of documentation and marketing, but few actual developers & productive applications using it
